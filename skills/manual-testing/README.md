# Manual Testing Skill

## 📋 Overview

Manual testing is the foundation of QA. This skill covers writing comprehensive test cases, executing tests, documenting results, and reporting bugs professionally.

## 🎯 When to Use This Skill

Use manual testing when:
- Building a test case suite from requirements
- Testing complex user workflows
- Need exploratory testing
- Validating UI/UX
- Testing before automation
- User acceptance testing (UAT)

## 📚 What You'll Learn

- Test case design and documentation
- Test execution best practices
- Result tracking and reporting
- Bug documentation standards
- Test metrics and coverage

## 🔄 Related Skills

- **Test Management** - Planning and organizing tests
- **Bug Reporting** - How to document issues
- **Test Data** - Creating test data for scenarios

## 📖 Framework

### 1. Test Case Design

**Structure:**
Test ID: TC-001
Feature: Login
Title: Valid user login
Preconditions: - User account exists
- User is on login page
Steps: 1. Enter email
2. Enter password
3. Click login
Expected: User logged in
Status: PASS/FAIL
### 2. Test Execution

**Process:**
1. Read test case
2. Set up preconditions
3. Execute each step
4. Compare actual vs expected
5. Document results
6. Mark status
