# Bug Reporting Skill

## 📋 Overview

Professional bug reporting is critical for developers to understand and fix issues quickly. This skill covers writing clear, actionable bug reports that get resolved faster.

## 🎯 When to Use This Skill

Use bug reporting when:
- Testing finds an issue
- Need to communicate with developers
- Tracking defects in Jira/Azure DevOps
- Want faster bug resolution
- Building team communication

## 📚 What You'll Learn

- Bug report structure
- Clear description writing
- Severity/priority assignment
- Reproduction steps
- Expected vs actual results
- Attachment and evidence
- Communication best practices

## 🔄 Related Skills

- **API Testing** - API bugs
- **Test Automation** - Automated bug detection
- **Test Management** - Bug tracking and triage

**Severity** = How bad is the bug?
- **Critical:** App crashes, data loss, security
- **High:** Major feature broken
- **Medium:** Feature partially broken
- **Low:** Minor issue, cosmetic

**Priority** = When fix it?
- **Critical:** Immediately
- **High:** This sprint
- **Medium:** Next sprint
- **Low:** Backlog

**Your Role:** Create great bug reports

## ✅ Checklist

Before submitting bug report:
- [ ] Title is clear and specific
- [ ] Severity/Priority assigned correctly
- [ ] Steps to reproduce are exact
- [ ] Expected result clearly stated
- [ ] Actual result described
- [ ] Environment documented
- [ ] Screenshots/logs attached
- [ ] No duplicate issues
- [ ] Bug is reproducible
- [ ] Actionable for developer

## 🚀 Best Practices

**DO:**
- ✅ One bug per report
- ✅ Be specific and clear
- ✅ Include exact error messages
- ✅ Provide screenshots/videos
- ✅ Use proper severity levels
- ✅ Include environment details
- ✅ Follow team's bug template

**DON'T:**
- ❌ Mix multiple bugs in one report
- ❌ Use vague descriptions
- ❌ Blame developers
- ❌ Forget attachments
- ❌ Misassign severity
- ❌ Report already-known issues
- ❌ Write novels (be concise)
