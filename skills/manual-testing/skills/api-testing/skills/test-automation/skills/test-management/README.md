# Test Management Skill

## 📋 Overview

Test management covers planning, organizing, executing, and reporting on testing activities. This is the foundation skill that all other testing skills build upon.

## 🎯 When to Use This Skill

Use test management when:
- Planning a testing strategy
- Organizing test execution
- Tracking test metrics
- Creating test plans
- Managing team testing efforts
- Reporting test status

## 📚 What You'll Learn

- Test planning and strategy
- Test scope definition
- Resource allocation
- Schedule planning
- Risk-based testing
- Test metrics and KPIs
- Status reporting
- Team coordination

## 🔄 Related Skills

All other skills depend on test management:
- **Manual Testing** - Organized via test plan
- **API Testing** - Tracked in test management
- **Test Automation** - Coordinated by test manager
- **Performance Testing** - Scheduled via plan

## 📖 Framework

### 1. Test Planning

**Test Plan Components:**
- Objective: What are we testing?
- Scope: What features/modules?
- Strategy: How will we test?
- Schedule: When will we test?
- Resources: Who will test?
- Risks: What could go wrong?
- Exit Criteria: When are we done?

## 🧪 Test Types Managed

- **Smoke Testing** - Quick sanity checks
- **Functional Testing** - Features work as designed
- **Regression Testing** - Previous features still work
- **Integration Testing** - Systems work together
- **UAT** - User acceptance testing
- **Performance Testing** - Load and scalability

## ✅ Checklist

Before test execution:
- [ ] Test plan created and approved
- [ ] Scope clearly defined
- [ ] Resources allocated
- [ ] Schedule realistic
- [ ] Risks identified
- [ ] Metrics defined
- [ ] Tools configured
- [ ] Team trained
